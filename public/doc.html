Inserir medição via TCP/UDP (descontinuada)

Conectar via TCP na porta 8008 do servidor e enviar as medições em formato string. A cada medição enviada, será retornado o valor do turno configurado.

Exemplo da string para envio de medição (pode ser enviado um "enter" para simbolizar fim da linha): 

&ch_id=VALOR&mc_cd=VALOR&field1=VALOR&field2=VALOR&field3=VALOR&field4=VALOR&field5=VALOR\n\r

Obs: A ordem dos parâmetros não interfere.

Parâmetro Tipo Obrigatório Descrição
&ch_id Inteiro Sim Código do canal. Ignora a transação caso não informado.
&mc_cd String Sim Código da máquina (pode ser enviado null, porém a medição ficará "orfã" de máquina)
&field1 String Não Campo 1 para envio de medição.
&field2 Float Não Campo 2 para envio de medição. Precisão de duas casas decimais.
&field3 Float Não Campo 3 para envio de medição. Precisão de duas casas decimais.
&field4 Float Não Campo 4 para envio de medição. Precisão de duas casas decimais.
&field5 String Não Campo 5 para envio de medição.

Inserir medição via Web API

Insere medições no canal vinculado ao token informado, se ocorrer sucesso na requisição, retorna o ID gerado à medicação.
Retorna erro 400 caso haja alguma exceção não tratada. 
/api/feed/update?token=VALOR&mc_cd=VALOR&field1=VALOR&field2=VALOR&field3=VALOR&field4=VALOR&field5=VALOR

Obs: A ordem dos parâmetros não interfere.

Parâmetro Tipo Obrigatório Descrição
&token String Sim Chave de autenticação vinculada ao canal. Retorna zero caso não informado ou incorreto.
&mc_cd String Sim Código da máquina (pode ser enviado null, porém a medição ficará "orfã" de máquina)
&field1 String Não Campo 1 para envio de medição.
&field2 Float Não Campo 2 para envio de medição. Precisão de duas casas decimais.
&field3 Float Não Campo 3 para envio de medição. Precisão de duas casas decimais.
&field4 Float Não Campo 4 para envio de medição. Precisão de duas casas decimais.
&field5 String Não Campo 5 para envio de medição.

Consultar medição

Retorna as medições inseridas em formato JSON conforme filtros enviados como parâmetros.
Retorna erro 400 caso haja alguma exceção não tratada. 
/api/CODIGO_CANAL/get?token=VALOR&fields=VALOR&results=VALOR&toXml=VALOR
Parâmetro Tipo Obrigatório Descrição
&token String Sim Chave de autenticação do canal. Retorno erro 404 caso não informado ou incorreto.
&fields String Não Quantidade de campos que deseja selecionar para retorno. Atualmente são de 1 a 5. Devem ser informados separados por virgula (ex:1,2,3,4,5). Se caso não informado, a API não retornará dados de medição.
&results Inteiro Não Quantidade de registros que deseja que a API retorno. Se não passar valor, a API entenderá que deseja receber todos as medições.
&toXml Inteiro Não Formato de retorno dos dados.
1 = XML
0 = JSON
Caso não passado o parâmetro, considera retorno padrão como JSON.

Deletar medições

Deleta medições do canal mantendo uma cópia de cada registro deletado em uma tabela de log. Permite passar um range de data e hora para deletar registros específicos. Caso o range nao for especificado, todos os registros do canal serão deletados. 
/api/CODIGO_CANAL/delete?token=VALOR&initial_date=99/99/9999 99:99&final_date=99/99/9999 99:99
Parâmetro Tipo Obrigatório Descrição
&token String Sim Chave de autenticação do canal. Retorno erro 404 caso não informado ou incorreto.
&initial_date String Não Data inicial para deletar os registros. Formato 99/99/9999 99:99
&final_date String Não Data final para deletar os registros. Formato 99/99/9999 99:99

Cadastrar máquina

Exibe cadastro de máquinas. Obrigatório informar token, código da máquina e nome. 
/api/machine

Configurar turno

Configura o turno do canal.

/api/CODIGO_CANAL/timeShift?token=VALOR&total=VALOR
Parâmetro Tipo Obrigatório Descrição
&token String Sim Chave de autenticação do canal. Retorna erro 404 caso não informado ou incorreto.
&total Inteiro Sim Valor do turno.

Última medição

Obter última medição da máquina.

/api/CODIGO_MAQ/getMax?token=VALOR&fields=VALOR
Parâmetro Tipo Obrigatório Descrição
&token String Sim Chave de autenticação do canal. Retorna erro 404 caso não informado ou incorreto.
&fields String Não Quantidade de campos que deseja selecionar para retorno. Atualmente são de 1 a 5. Somente numero deverá ser informado (ex: &fields=1). Se caso não informado, a API retornará um valor nulo.

Inserir pausa via IoT

Inserir pausa via IoT.

/api/machinepause?token=VALOR&mc=VALOR&p=VALOR&sd=VALOR&ed=VALOR&pr=VALOR
Parâmetro Tipo Obrigatório Descrição
&token String Sim Chave de autenticação do canal. Retorna erro 404 caso não informado ou incorreto.
&mc String Sim Código da máquina
&p Inteiro Sim Pausa em minutos
&sd String Sim Data inicial da pausa
&ed String Sim Data final da pausa
&pr Inteiro Sim Id do motivo da pausa
